#  Bufs Book Review
釜山外大の学生たちが読んだ本の感想を共有し、おすすめし合うWebサービスです。
「本を読むのが好きで、他の学生がどんな本を読んでいるか知りたい」という思いから企画・開発しました。

## プロジェクト概要
- **開発期間:** 9月 ~ 12月 (週6時間程度)
- **開発人数:** 1名 (個人プロジェクト)
- **デプロイURL:** https://bufs-book-review-2.onrender.com

## 使用技術
- **Frontend:** HTML, CSS, JavaScript (モバイル対応レスポンシブデザイン)
- **Backend:** Node.js
- **Database & Auth:** Firebase (Firestore, Authentication)
- **API:** NAVER API (本検索機能)
- **Infra/Hosting:** Render (PaaS), GitHub Actions (自動デプロイ), UptimeRobot

## 主な機能
1. **本検索機能:** NAVER APIと連携し、レビューしたい本を簡単に検索可能。
2. **レビュー＆コミュニケーション:** Firestoreを利用し、感想の投稿だけでなく、他人の感想への「いいね」や「返信」ができる機能。
3. **時間帯別の動的UI:** ユーザーのアクセス時間帯に合わせて、背景（図書館の風景など）が自動で変化する機能。
4. **安全なユーザー認証:** Firebase Authenticationを利用し、存在しないメールアドレスでの不正登録を防止。

## 工夫した点・トラブルシューティング
本プロジェクトでは、開発中に直面した課題を以下のように解決し、完成度を高めました。

**1. 初めてのバックエンド構築とサーバー待機状態の解決**
NAVER APIを利用するため、初めてNode.jsでバックエンドサーバーを構築しました。初期はエラーが多発しましたが、原因を分析し実装を完了しました。また、Renderの無料プランによる「サーバーのスリープ問題」が発生したため、**UptimeRobot**を導入してスリープを防止し、シームレスな通信環境を維持するよう工夫しました。

**2. フィードバックを活かしたUI/UX改善**
中間発表での青海進の仲間や先生からのフィードバックを積極的に取り入れました。
- **機能拡張:** 単に感想を書くだけでなく、時間帯による背景変更機能や、「いいね」「返信」機能を追加し、ユーザー間のコミュニケーションを活性化させました。
最初は自分のサイトを見せることに抵抗がありましたが、第三者の意見を取り入れることで、自分では気づけなかった不便な点に気づき、プロダクトの質を大幅に向上させることができました。
